# FastVox 开发者测试指南

本指南旨在指导开发者对 FastVox 实现全方位的技术测试，确保分布式推理架构的稳定性、低延迟特性及 Zero-shot 声音克隆的质量。

---

## 1. 测试环境准备

在开始测试前，请确保满足以下条件：
- **Python**: 3.10+ 环境已激活。
- **依赖**: 已执行 `pip install -r requirements.txt`。
- **模型**: `models/zipvoice/` 下已存放 `encoder.onnx`, `decoder.onnx`, `vocos_24khz.onnx`。
- **工具**: 推荐使用 `Postman` (REST) 和 `wscat` (WebSocket) 进行接口分析。

---

## 2. 单元与组件测试 (Component Level)

### 2.1 文本切分器 (TextSplitter)
验证长文本是否能在标点符号处正确断句，且单个分片长度不超过 200 字。
- **测试方法**: 运行 `app/inference/text_splitter.py` 中的单元测试。
- **关键点**: 检查“。”、“！”、“，”及换行符的边界处理逻辑。

### 2.2 共享内存 (SHMManager)
验证主进程与子进程能否通过物理内存地址进行零拷贝读写。
- **测试方法**:
  1. 主进程调用 `shm_manager.create()`。
  2. 模拟写入 2MB 随机字节。
  3. 子进程调用 `shm_manager.attach()` 并读取，对比 MD5 哈希值。

### 2.3 UDS 信令协议 (UDSProtocol)
验证指令在 1ms 内完成跨进程投递。
- **测试方法**: 启动 `UDSServer`，使用 `UDSClient` 快速发送 1000 条 READY 信令。
- **关键点**: 检查信令字段 `slot_id` 和 `status` 的解析准确度。

## 3. 集成与 API 测试 (API Level)

### 3.1 身份认证测试
- **接口**: `POST /auth/jwt/login`
- **验证**: 获取 `access_token`，并验证其在后续 `Authorization: Bearer <token>` 中的有效性。

### 3.2 声纹上传与归一化
- **接口**: `POST /api/v1/voice/upload`
- **验证**:
  1. 上传一个 44.1kHz 的 MP3。
  2. 检查 `data/voices/` 下生成的 WAV 文件是否已降采样为 **24kHz Mono**。
  3. 检查时长验证（小于 3s 或大于 30s 是否触发 400 错误）。

### 3.3 WebSocket 实时推理测试 (核心)
- **接口**: `WS /api/v1/tts/stream?token=<jwt>`
- **操作逻辑**:
  1. 连接成功后发送 JSON: `{"text": "你好世界", "voice_id": "xxx"}`。
  2. **观察**: 应该是先收到一串 Binary 帧（Opus 数据），最后收到一个 0 字节帧（结束符）。
  3. **指标**: 记录从发送 JSON 到收到第一个 Binary 帧的时间（首字耗时，建议 < 500ms）。

## 4. 压力与并发测试 (Performance Level)

### 4.1 并发上限测试 (Slot Limit)
- **并发策略**: 默认 `SLOT_COUNT=5`。
- **方法**: 使用多个客户端同时发起 10 个请求。
- **预期结果**: 前 5 个请求立即进入推理，后 5 个请求应在 `SlotManager` 中通过信号量排队，而不是直接报错。

### 4.2 长效稳定性测试
- **场景**: 模拟 2 小时持续合成。
- **观察**:
  - `psutil` 监控主进程及 Worker 的 RSS 内存占用是否溢出。
  - 检查 `/run/fastvox/` 或临时目录下的 `.sock` 文件是否泄露。
  - 观察 `watchdog` 协程是否能正确回收僵死槽位。

## 5. UI/UX 冒烟测试 (Frontend Level)

### 5.1 响应式布局
- **测试**: 缩小浏览器窗口，查看侧边栏是否自动收起，文字输入框是否保持可用。

### 5.2 状态反馈
- **测试**: 点击“开始合成”按钮。
- **验证**: 
  1. 按钮进入 Loading 状态。
  2. 右侧播放器区域显示 **Skeleton (骨架屏)** 脉冲动画。
  3. 合成完成后音频波形显示。

---

## 6. 开发者调试技巧 (Debug Tips)

- **Worker 日志**: Worker 是独立进程，日志输出可能被缓冲或重定向，建议检查 `stdout`。
- **SHM 残留**: 如果程序异常崩溃导致下次启动报错 `FileExistsError`，请手动执行 `scripts/cleanup_shm.sh`。
- **UDS 路径**: 在 Windows 上，确保 `TEMP` 路径不包含空格或非 ASCII 字符，否则可能导致 Socket 绑定失败。
