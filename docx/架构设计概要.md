## 1. 项目概述

### 1.1 项目定位

**FastVox** 是一款面向轻量化硬件的高性能 TTS（文字转语音）服务系统。基于 `sherpa-onnx` 推理引擎和 `zipvoice` 零射声音克隆模型，通过 **"信令与数据分离"** 架构，在 **2GB 内存** 限制下实现低延迟、高并发的多端语音合成服务。

### 1.2 支持平台

| 平台 | 硬件参考 | 优先级 |
| :--- | :--- | :--- |
| **macOS (ARM64)** | Mac Mini M4 (8GB RAM) | 主力部署 |
| **Linux (x86_64 / ARM64)** | Ubuntu 22.04+ / Debian 12+ | 生产部署 |

### 1.3 核心能力

- 零射声音克隆 (Zero-Shot Voice Cloning)
- WebSocket 实时流式语音合成
- HTTP Chunked Transfer 兼容微信小程序
- 多用户并发调度 (Slot 机制)
- 声纹特征管理 (Speaker Embedding)

## 2. 核心技术栈

| 层级 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| **后端框架** | Python 3.10+ / FastAPI | Single Worker 模式，uvicorn 驱动 |
| **推理引擎** | sherpa-onnx (pip) | 跨平台，支持 macOS ARM64 和 Linux x86_64/ARM64 |
| **TTS 模型** | zipvoice (distill-int8) | 零射声音克隆，encoder + decoder + vocoder 三件套 |
| **进程通信 - 数据平面** | `multiprocessing.shared_memory` | Ring Buffer 模式的共享内存 |
| **进程通信 - 控制平面** | Unix Domain Socket (UDS) | 异步信令，指针同步 |
| **音频编码** | PyAV | 内存级 PCM → Opus/Ogg 编码（禁止 fork ffmpeg 子进程） |
| **Web 前端** | Vue 3 + Vite + Vanilla CSS | 管理后台 + 语音交互界面 (飞书风格极简设计) |
| **权限管理** | FastAPI Users | JWT (Web) + code2Session (微信小程序) |
| **数据库** | SQLite + SQLModel | 存储用户、声纹特征、使用记录 |
| **进程管理** | Supervisor (Linux) / launchd (macOS) | 守护进程管理 |
| **反向代理** | Nginx / Caddy | TLS 终止 + WebSocket 代理 |

### 3.2 数据流

```
用户请求文本 → Gateway 鉴权 → SlotManager 分配 Slot
    → UDS 下发任务指令 → Worker 加载 Speaker Embedding
    → sherpa-onnx 推理生成 PCM → 写入 SHM Slot
    → UDS 回报 (offset, size, status) → Gateway 读取 SHM
    → PyAV 编码 Opus/Ogg → WebSocket/HTTP 推送客户端
```

## 5. 核心模块设计

### 5.1 平台兼容层 (`platform_compat.py`)

**设计目标**: 抽象 macOS 和 Linux 的系统级差异，对上层模块透明。

```python
# 伪代码 - 关键差异点

class PlatformConfig:
    """根据运行平台自动选择最优配置"""
    
    @property
    def shm_path(self) -> str:
        """SHM 文件系统路径"""
        if sys.platform == "darwin":
            return "/tmp"       # macOS SHM 通过 POSIX shm_open
        return "/dev/shm"       # Linux 直接访问 tmpfs
    
    @property
    def uds_path(self) -> str:
        """UDS socket 文件路径"""
        if sys.platform == "darwin":
            return "/tmp/fastvox_worker_{}.sock"
        return "/run/fastvox/worker_{}.sock"
    
    @property
    def process_start_method(self) -> str:
        """多进程启动方式"""
        if sys.platform == "darwin":
            return "spawn"      # macOS 强制 spawn
        return "forkserver"     # Linux 使用 forkserver 减少内存拷贝
    
    @property
    def max_shm_size(self) -> int:
        """SHM 最大尺寸 (字节)"""
        # macOS SHM 默认限制较小，需检查 sysctl kern.sysv.shmmax
        if sys.platform == "darwin":
            return 10 * 1024 * 1024   # 10MB
        return 10 * 1024 * 1024       # 10MB

    @property
    def recommended_workers(self) -> int:
        """推荐 Worker 数量"""
        if sys.platform == "darwin":
            return 2    # M4 高效核心数量考量
        return min(os.cpu_count() or 2, 4)  # Linux 上适当增加
```

**macOS vs Linux 关键差异**:

| 维度 | macOS (darwin) | Linux |
| :--- | :--- | :--- |
| 多进程启动 | `spawn` (强制) | `forkserver` (推荐) |
| SHM 路径 | POSIX `shm_open` (映射 /tmp) | `/dev/shm` (tmpfs) |
| UDS 路径 | `/tmp/fastvox_*.sock` | `/run/fastvox/*.sock` |
| SHM 清理 | `shm.unlink()` | `shm.unlink()` + 清扫 `/dev/shm/fastvox_*` |
| 守护进程 | launchd (plist) | systemd / Supervisor |
| 信号处理 | 支持 SIGTERM/SIGINT | 额外支持 SIGUSR1 (日志轮转) |
| ARM 优化 | Apple Neural Engine (ANE) 可选 | ARM NEON (aarch64) 自动启用 |

### 5.2 共享内存管理 (`shm_manager.py`)

```python
# 伪代码 - SHM Ring Buffer 设计

SHM_NAME = "fastvox_shm_main"
SLOT_COUNT = 5
SLOT_SIZE = 2 * 1024 * 1024  # 2MB per slot
TOTAL_SHM_SIZE = SLOT_COUNT * SLOT_SIZE  # 10MB

class SHMManager:
    """
    职责:
    1. Lifespan 启动时创建/附加 SHM
    2. 提供 slot_id → (offset, buffer_view) 映射
    3. Lifespan 关闭时 unlink 清理
    """
    
    def create(self) -> None:
        """创建 SHM 块，若已存在则先清理"""
        self._cleanup_stale()
        self.shm = shared_memory.SharedMemory(
            name=SHM_NAME, create=True, size=TOTAL_SHM_SIZE
        )
    
    def get_slot_buffer(self, slot_id: int) -> memoryview:
        """获取指定 Slot 的内存视图 (零拷贝)"""
        offset = slot_id * SLOT_SIZE
        return memoryview(self.shm.buf[offset:offset + SLOT_SIZE])
    
    def destroy(self) -> None:
        """安全释放"""
        self.shm.close()
        self.shm.unlink()
    
    def _cleanup_stale(self) -> None:
        """平台感知的残留清理"""
        if sys.platform == "linux":
            # 直接扫描 /dev/shm/fastvox_*
            for f in Path("/dev/shm").glob("fastvox_*"):
                f.unlink(missing_ok=True)
        else:
            # macOS: 尝试附加后 unlink
            try:
                stale = shared_memory.SharedMemory(name=SHM_NAME, create=False)
                stale.close()
                stale.unlink()
            except FileNotFoundError:
                pass
```

### 5.3 UDS 信令协议 (`uds_protocol.py`)

```python
# 信令帧格式 (16 字节定长)
# struct.pack('IIII', slot_id, offset, size, status)
#
# status 枚举:
#   0 = IDLE        - 空闲
#   1 = WRITING     - 正在写入
#   2 = READY       - 数据就绪，可读取
#   3 = ERROR       - 推理出错
#   4 = HEARTBEAT   - Worker 心跳
#   5 = COMPLETE    - 整句推理完成

SIGNAL_FORMAT = 'IIII'
SIGNAL_SIZE = struct.calcsize(SIGNAL_FORMAT)  # 16 bytes

class UDSServer:
    """主进程端 - 监听 Worker 信令"""
    
    async def start(self, loop: asyncio.AbstractEventLoop):
        """绑定 UDS 并注册到 asyncio 事件循环"""
        self.sock = socket.socket(socket.AF_UNIX, socket.SOCK_DGRAM)
        self.sock.bind(self.platform.uds_path.format("gateway"))
        self.sock.setblocking(False)
        loop.add_reader(self.sock.fileno(), self._on_signal)
    
    def _on_signal(self):
        """收到信令时回调"""
        data = self.sock.recv(SIGNAL_SIZE)
        slot_id, offset, size, status = struct.unpack(SIGNAL_FORMAT, data)
        # 派发到对应的 Slot 处理协程
        asyncio.ensure_future(self._dispatch(slot_id, offset, size, status))

class UDSClient:
    """Worker 端 - 发送信令"""
    
    def send_signal(self, slot_id: int, offset: int, size: int, status: int):
        """向 Gateway 发送定长信令"""
        data = struct.pack(SIGNAL_FORMAT, slot_id, offset, size, status)
        self.sock.sendto(data, self.gateway_addr)
```

### 5.5 TTS 引擎封装 (`tts_engine.py`)

```python
class TTSEngine:
    """
    封装 sherpa-onnx 推理，运行在 Worker 子进程中。
    
    关键配置:
    - zipvoice encoder/decoder/vocoder 三件套
    - intra_op_num_threads=1 (禁止线程并发)
    - ORT Arena 内存限制
    """
    
    def __init__(self, model_dir: str, platform: PlatformConfig):
        import sherpa_onnx
        
        tts_config = sherpa_onnx.OfflineTtsConfig(
            model=sherpa_onnx.OfflineTtsModelConfig(
                zipvoice=sherpa_onnx.OfflineTtsZipVoiceModelConfig(
                    encoder=f"{model_dir}/encoder.onnx",
                    decoder=f"{model_dir}/decoder.onnx",
                    vocoder=f"{model_dir}/vocos_24khz.onnx",
                    tokens=f"{model_dir}/tokens.txt",
                    lexicon=f"{model_dir}/lexicon.txt",
                    data_dir=f"{model_dir}/espeak-ng-data",
                    num_steps=4,  # distill 模型步数 (快速)
                ),
                num_threads=1,    # 单线程策略
                debug=False,
                provider="cpu",   # macOS/Linux 通用
            ),
            max_num_sentences=1,
        )
        self.tts = sherpa_onnx.OfflineTts(tts_config)
    
    def synthesize(
        self,
        text: str,
        speaker_embedding: list[float] | None = None,
        prompt_audio_path: str | None = None,
        prompt_text: str | None = None,
        speed: float = 1.0,
    ) -> tuple[bytes, int]:
        """
        执行推理，返回 (pcm_bytes, sample_rate)
        PCM 格式: int16, mono
        """
        audio = self.tts.generate(
            text,
            sid=0,
            speed=speed,
        )
        # 转 int16 PCM
        import numpy as np
        pcm = (np.array(audio.samples) * 32767).astype(np.int16)
        return pcm.tobytes(), audio.sample_rate
    
    def warmup(self):
        """预热推理引擎 (空跑一次消除冷启动延迟)"""
        self.synthesize("你好", speed=1.0)
```

### 5.6 音频编码器 (`audio_encoder.py`)

```python
class AudioEncoder:
    """
    使用 PyAV 在内存中将 PCM 编码为 Opus/Ogg。
    
    关键要求:
    - 全程在内存中完成，严禁调用外部 ffmpeg 进程
    - 支持流式编码 (接收 PCM 分片，输出 Ogg 分片)
    """
    
    def encode_opus_chunk(
        self,
        pcm_data: bytes,
        sample_rate: int = 24000,
        channels: int = 1,
    ) -> bytes:
        """将 PCM 分片编码为 Ogg/Opus 容器"""
        import av
        
        output_buf = io.BytesIO()
        output_container = av.open(output_buf, mode='w', format='ogg')
        stream = output_container.add_stream('libopus', rate=sample_rate)
        stream.channels = channels
        
        # 将 PCM bytes 转为 AudioFrame
        frame = av.AudioFrame.from_ndarray(
            np.frombuffer(pcm_data, dtype=np.int16).reshape(1, -1),
            format='s16',
            layout='mono',
        )
        frame.sample_rate = sample_rate
        
        for packet in stream.encode(frame):
            output_container.mux(packet)
        for packet in stream.encode(None):  # flush
            output_container.mux(packet)
        
        output_container.close()
        return output_buf.getvalue()
```

### 5.7 Worker 子进程入口 (`worker_main.py`)

```python
def worker_main(
    worker_id: int,
    shm_name: str,
    uds_gateway_addr: str,
    model_dir: str,
):
    """
    Worker 子进程的主函数。
    
    生命周期:
    1. 附加 SHM
    2. 初始化 TTSEngine (sherpa-onnx)
    3. 预热推理
    4. 循环等待 UDS 指令
    5. 执行推理 → 写入 SHM Slot → 发送信令
    """
    
    # 1. 平台感知设置
    platform = PlatformConfig()
    
    # 2. 附加共享内存
    shm = shared_memory.SharedMemory(name=shm_name, create=False)
    
    # 3. 初始化推理引擎
    engine = TTSEngine(model_dir, platform)
    engine.warmup()
    
    # 4. 初始化 UDS 客户端
    uds = UDSClient(uds_gateway_addr)
    
    # 5. 主循环 - 监听分配的任务
    while True:
        task = receive_task()  # 通过 UDS 或 Queue 接收任务
        slot_id = task.slot_id
        
        try:
            pcm_data, sample_rate = engine.synthesize(
                text=task.text,
                speaker_embedding=task.speaker_embedding,
            )
            
            # 写入 SHM Slot
            slot_offset = slot_id * SLOT_SIZE
            shm.buf[slot_offset:slot_offset + len(pcm_data)] = pcm_data
            
            # 发送就绪信令
            uds.send_signal(slot_id, slot_offset, len(pcm_data), STATUS_READY)
            
        except Exception as e:
            uds.send_signal(slot_id, 0, 0, STATUS_ERROR)
```

## 10. 安全策略

| 策略 | 实现方式 |
| :--- | :--- |
| **文本长度熔断** | API 入口层校验 `len(text) ≤ 500`，超出直接返回 422 |
| **IP 频率限制** | 令牌桶算法，默认 30 req/min，超出返回 429 |
| **Slot 超时熔断** | Watchdog 协程，Slot 占用 > 30s 强制释放 |
| **SHM 泄漏防御** | Lifespan shutdown 清理 + 启动前自愈扫描 |
| **音频文件清理** | 声纹提取后立即删除原始音频，仅保留 Embedding |
| **JWT 安全** | 生产环境必须配置独立密钥，默认 1h 过期 |
